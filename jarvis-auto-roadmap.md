# Годовая дорожная карта: Jarvis Auto MVP → Экосистема

## Этап 0: Подготовка и инфраструктура (Месяц 1)

**Цель:** заложить технический фундамент и определить бизнес-модель до написания кода.

### Неделя 1-2: Анализ и стратегия (120 часов)

- **День 1-3 (24 часа)** — Определение целевого региона (выберите 1 город с населением до 500 тыс. для пилота)
- **День 4-5 (16 часов)** — Анализ конкурентов (Autodoc, Drive2, exist.ru, local автосервисы)
- **День 6-7 (16 часов)** — Опрос 30-50 автовладельцев: какие поломки случаются чаще, уровень доверия к приложениям
- **День 8-10 (24 часа)** — Составление карты пользовательских сценариев (CJM) для 3 типов: новичок, опытный, мастер
- **День 11-14 (40 часов)** — Выбор стека (Firebase + Flutter), настройка репозиториев, CI/CD

### Неделя 3-4: Прототип дизайна и бэкенда (120 часов)

- **День 15-17 (24 часа)** — Дизайн-система: компоненты Deep Blue и Safety Orange, типографика, иконки
- **День 18-21 (40 часов)** — Figma-прототип главного экрана: дашборд с фарой-кнопкой и шкалой здоровья
- **День 22-24 (32 часа)** — Настройка Firebase: Firestore, Authentication, Storage
- **День 25-28 (24 часа)** — Юридическая подготовка: оферта, политика конфиденциальности

**Итог этапа:** Дизайн-система, прототип в Figma, настроенный Firebase, юридические документы.

## Технический стек

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| Мобильный клиент | Flutter 3 | Один код → Android + iOS, быстрый рендеринг |
| Бэкенд / БД | Firebase (Firestore + Functions + FCM) | Serverless, бесплатный тир на старте |
| ИИ-диагностика | Claude API (Sonnet) через Cloud Functions | Ключ не светится на клиенте |
| Карты и геолокация | 2GIS SDK (РФ-приоритет) + Yandex Maps | Полнота данных по российским городам |
| Оплата | ЮKassa + Google Play Billing | Поддержка карт РФ, App Store требования |
| Аналитика | Firebase Analytics + BigQuery | Бесплатная воронка, мощный SQL |
| CI/CD | GitHub Actions + Fastlane | Автодеплой в Play Store по тегу |
